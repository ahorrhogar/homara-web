## Anatomía estándar de un artículo

Cada `.md` seguirá esta plantilla (markdown puro, sin frontmatter):

```
# Título H1 con keyword principal

> Lede de 2-3 frases con la promesa editorial y para quién es este artículo.

## Resumen rápido (TL;DR)
Lista de 4-6 puntos.

## [Bloques específicos del tipo de artículo]
- Comparativas: tabla resumen + análisis criterio a criterio + casos de uso.
- Guías: factores a considerar + errores comunes + decisión final.
- Rankings: tabla resumen + ficha por producto (3-4 párrafos + pros/contras + CTA placeholder).
- Pillars: hub con secciones extensas + cards a satélites.

## Recomendaciones por perfil
"Si buscas X → producto Y" (3-4 perfiles).

## Errores comunes al comparar [keyword]
3-5 errores frecuentes con explicación.

## Preguntas frecuentes
5-8 preguntas reales (FAQPage schema).

## Artículos relacionados
- [Título relacionado 1](slug-1)
- [Título relacionado 2](slug-2)
- [Título relacionado 3](slug-3)

## JSON-LD (para integrar en TSX)
```json
{ ... FAQPage + Article + BreadcrumbList ... }
```
```

CTAs en cuerpo (1 por producto destacado + 1 secundaria a `/comparador` o `/categoria/<slug>` al final): `[Ver oferta]({{OFFER:slug-producto}})`.
